

# Stock Volatility Prediction Project
**A 30-Second Forecast Framework for Options Market Making**

This project contains data processing notebooks, machine learning models, stock analysis notebooks, and a Streamlit dashboard for visualisation and prediction.

DATA3888 Capstone - University of Sydney

---

# Required Packages

Install the required packages before running the project:

```bash
pip install streamlit pandas numpy plotly joblib notebook matplotlib seaborn scipy scikit-learn statsmodels xgboost
```


---

# Files

## 1. `01_data_processing.ipynb`

### What is it?

This notebook:

* Cleans and processes the raw stock/order book data
* Creates engineered features
* Generates datasets used for modelling

### How to run

Open the notebook and run all cells.

---

## 2. `02_modelling_temporal.ipynb`

### What is it?

This notebook:

* Trains machine learning models
* Performs temporal validation
* Evaluates volatility prediction performance

### Files needed

Requires output files generated from:

* `01_data_processing.ipynb`

### How to run

Open the notebook and run all cells.

---

## 3. `03_Tech_stocks.ipynb`

### What is it?

This notebook:

* Performs analysis on technology stocks
* Generates additional comparisons and visualisations

### Files needed

Requires outputs/models generated from:

* `02_modelling_temporal.ipynb`

### How to run

Open the notebook and run all cells.

---

## 4. `Dashboard.py`

### What is it?

This is the Streamlit dashboard for:

* Viewing stock volatility predictions
* Exploring visualisations and analytics
* Interacting with model outputs

### Files needed

Requires:

* Generated datasets
* Saved models
* Dashboard output files from previous notebooks

The dashboard automatically searches folders such as:

* `models/`
* `dashboard_outputs/`
* `app_artifacts/`


### How to run

Run this command in terminal:

```bash
streamlit run "Dashboard.py"
```

After running, open the local link shown in terminal.
Note: It might take a few seconds for the dashboard to start. 

---

## 5. `Report.ipynb`
### What is it?
A self-contained HTML report notebook covering:
* Methodology and data pipeline summary
* Model performance tables (QLIKE, RMSE, RMSPE) across all horizons
* Generalisation analysis across seen, unseen, and named tech stocks
* Feature importance visualisation
* Appendix: full feature lists, hyperparameter tables, and RMSE/RMSPE results

### Files needed

All required files are pre-generated by the previous notebooks. The report does **not** require retraining any models. Requires the `dashboard_outputs/` and `models/` folders to be present with outputs from the previous notebooks, and `Workflow.png` and `Dashboard.png` in the same directory as the report.


### How to run
Ensure the `dashboard_outputs/` and `models/` folders are present, then run:
```bash
quarto render report.ipynb
```


# Recommended Execution Order

Run the files in this order:

1. `01_data_processing.ipynb`
2. `02_modelling_temporal.ipynb`
3. `03_Tech_stocks.ipynb`
4. `Dashboard.py`

Can run report whenever, provided the necessary folders/files are there.
