        ### 🎵 What is a WAV File?

        **WAV** stands for **Waveform Audio File Format**. It’s a **standard audio file format** used for storing raw, uncompressed audio on PCs (especially Windows).

        Think of it like this: a WAV file is like a **digital sound recording** of audio, saved in a very **high quality**, without losing any details.

        ---

        ### 📦 What's Inside a WAV File?

        A WAV file is made up of two main parts:

        1. **Header** – tells the computer how to read the audio data  
        2. **Data** – the actual sound (the audio samples)

        Let’s go a bit deeper:

        #### 1. **Header**
        - Contains metadata: how long the audio is, sample rate, number of channels, bit depth, etc.
        - It’s about 44 bytes long for standard PCM WAV files.

        #### 2. **Audio Data**
        - This is the raw sound.
        - Stored as a series of numbers (samples) that represent the sound wave.

        ---

        ### 🎙️ Key Terms to Know

        | Term             | Meaning |
        |------------------|---------|
        | **Sample**       | One point in the sound wave (like a snapshot of the sound’s loudness at a specific moment). |
        | **Sample Rate**  | How many samples per second (e.g., 44,100 Hz = 44,100 samples/sec). |
        | **Bit Depth**    | How precise each sample is (e.g., 16-bit means each sample can have 65,536 values). |
        | **Channels**     | Mono = 1 channel, Stereo = 2 channels. |

        So for example:
        - A **44.1 kHz, 16-bit stereo** WAV file = CD quality audio

        ---

        ### 🔉 Why Use WAV?

        **Pros:**
        - High quality
        - Lossless (no data is lost)
        - Easy to edit

        **Cons:**
        - Large file size
        - Not compressed like MP3

        ---

        A **WAV file** is:
        - A digital audio file format
        - Stores high-quality sound
        - Contains a header + audio data
        - Often used in music production, sound design, and programs that need full-quality audio

    would we use wb or w when writing into a wav file? can we use fwrite to write into the file?
        Great question! Let's break it down:

        ---

        ### 📝 Should You Use `"w"` or `"wb"`?

        You should **always use `"wb"`** when writing to a **WAV file** in C.

        #### Why `"wb"`?

        - `"w"` = write in **text mode** (can mess with binary data, especially on Windows—newline characters may be changed automatically)
        - `"wb"` = write in **binary mode** (exact raw bytes are written)

        > WAV files are binary files, so **using `"wb"` ensures nothing gets corrupted**.

        ---

        ### ✅ Can You Use `fwrite`?

        **Yes! `fwrite` is the correct function to use** when writing binary data like WAV files.

        #### Example:
        ```c
        FILE *fp = fopen("output.wav", "wb");
        if (fp == NULL) {
            perror("Error opening file");
            return 1;
        }

        // Write header
        fwrite(&wav_header, sizeof(wav_header), 1, fp);

        // Write audio samples
        fwrite(audio_data, sizeof(int16_t), num_samples, fp);

        fclose(fp);
        ```

        In this code:
        - `wav_header` is a struct you define that contains WAV header info (like RIFF chunk, fmt chunk, etc.)
        - `audio_data` is an array of raw audio samples (usually `int16_t` for 16-bit WAV)

        ---

        |-------------------|---------------|

        | Function to write | `fwrite()`    |

    but if the head has 44 bytes would it be better to use write or memcpy? 
        Great follow-up! Let's clear up the difference and when to use what when dealing with the **44-byte WAV header**:

        ---

        ### 🔧 `memcpy` vs `fwrite` vs `write`

        | Function   | What it does | Used for | Notes |
        |------------|--------------|----------|-------|
        | `memcpy`   | Copies memory from one place to another in RAM | Building header in memory | Does **not** write to file |
        | `fwrite`   | Writes data from memory to a file (in C stdlib) | Writing binary data to file | Buffered I/O |
        | `write`    | Writes data to a file descriptor (low-level sys call) | More control, used in Linux systems | Unbuffered I/O |

        ---

        ### ✅ So What's Best for Writing the 44-byte Header?

        You should **use `fwrite`**, **not `memcpy`**, to write the 44-byte WAV header **to a file**.

        #### Why not `memcpy`?
        - `memcpy` only moves data **within memory** (RAM). It **does not write anything to disk**.
        - You would use `memcpy` *before* `fwrite`, for example, if you were assembling the WAV header in a buffer first.

        ---

        ### 🔄 Example: Building the Header in Memory First

        If you want full control, you could:

        ```c
        uint8_t header[44];
        memcpy(header, ...);  // copy RIFF chunk
        memcpy(header + 12, ...);  // fmt chunk
        memcpy(header + 36, ...);  // data chunk
        fwrite(header, sizeof(header), 1, file);
        ```

        Or more commonly, use a struct:

        ```c
        struct wav_header {
            char riff[4];      // "RIFF"
            uint32_t size;
            char wave[4];      // "WAVE"
            char fmt[4];       // "fmt "
            uint32_t fmt_size;
            uint16_t audio_format;
            uint16_t channels;
            uint32_t sample_rate;
            uint32_t byte_rate;
            uint16_t block_align;
            uint16_t bits_per_sample;
            char data[4];      // "data"
            uint32_t data_size;
        } header;

        fwrite(&header, sizeof(header), 1, file);
        ```

        ---

        ### 🧠 Summary

        - Use **`fwrite`** to write to the file.
        - Use **`memcpy`** to prepare or copy header parts *in memory*, but not for file output.
        - Only use **`write`** if you're working with low-level file descriptors.
